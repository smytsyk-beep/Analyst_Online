## 🔧 Технические детали

### Используемые технологии

- **Sanity Client** (`next-sanity`) — подключение к CMS
- **GROQ** — язык запросов Sanity
- **TypeScript** — типизация данных
- **Node.js fs** — работа с файловой системой
- **CSV** — формат экспорта

### Особенности реализации

#### 1. Конвертация Portable Text

Portable Text (формат Sanity) конвертируется в plain text:

- Блоки текста извлекаются и объединяются
- Изображения заменяются на `[Image: alt text]`
- Сохраняется читаемость контента

#### 2. Экранирование CSV

Корректная обработка специальных символов:

- Кавычки экранируются (`"` → `""`)
- Поля с запятыми/переносами оборачиваются в кавычки
- Поддержка многострочного текста

#### 3. JSON данные

Сложные структуры сохраняются как JSON строки:

- Метрики кейсов
- Теги и категории
- OG метаданные
- Дополнительные параметры (order, featured, icon и т.д.)

## 🚀 Как использовать

### Экспорт контента

```bash
# Запуск скрипта экспорта
npm run export-content
```

### Открытие CSV

- **Excel**: File → Open → выбрать CSV
- **Google Sheets**: File → Import → Upload → выбрать CSV
- **VS Code**: Установить расширение "Rainbow CSV"

### Фильтрация данных

#### По локали

```
Locale = "ru"  # только русский контент
Locale = "ua"  # только украинский
Locale = "ro"  # только румынский
```

#### По типу контента

```
Type = "service"        # только услуги
Type = "faq"           # только FAQ
Type = "omnidashBlock" # только блоки OmniDash
```

#### По наличию SEO

```
SEO Title != ""        # есть SEO заголовок
SEO Description = ""   # нет SEO описания
```

## 📝 Примечания

### Формат данных

- **Кодировка**: UTF-8
- **Разделитель**: запятая (`,`)
- **Перенос строки**: `\n`
- **Экранирование**: двойные кавычки (`""`)

### Ограничения

- Portable Text конвертируется в plain text (теряется форматирование)
- Изображения представлены только URL (не экспортируются файлы)
- Вложенные структуры сохраняются как JSON строки

### Безопасность

- Файл содержит только публичный контент
- API ключи и токены не экспортируются
- Файл можно безопасно передавать переводчикам и редакторам
